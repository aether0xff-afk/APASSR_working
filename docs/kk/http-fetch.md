# http-fetch KK reference

## 목적
- HTTP 요청을 수행하고 응답 메타데이터를 JSON 스키마로 수집합니다.

## 입력(Policy/KK)
- 파라미터
  - `METHOD`, `TIMEOUT`, `FOLLOW`, `MAX_REDIRECTS`, `USER_AGENT`, `HEADER`, `SCHEME`
  - `PATH`/`PATH_HINT`
- KK 입력(주요)
  - `PORT_LIST`, `TARGET_IP`, `PATH_HINT`

## 출력(KK 업데이트)
- HTTP 응답
  - `HTTP_STATUS_CODE`, `HTTP_STATUS_302`, `HTTP_HEADER_LOCATION_PRESENT`, `HTTP_HEADER_XNEXT_PRESENT`
  - `HTTP_TITLE`, `HTTP_CONTENT_TYPE`, `HTTP_CONTENT_LENGTH`, `HTTP_BODY_SHA1`, `HTTP_BODY_SNIPPET`
  - `HEADER_PRESENT`, `HTTP_SET_COOKIE`, `COOKIE_SEEN`, `REDIRECT_CHAIN_DEPTH`
- 구조/관측 힌트
  - `FORMS_DETECTED`, `PARAM_DETECTED`, `LINK_FOUND`, `FORM_ACTION`, `SCRIPT_SRC`, `JS_ENDPOINT`
  - `ROBOTS_DISALLOW`, `ROBOTS_ALLOW`, `SITEMAP_URL`, `PATH_HINT`, `PATH_SEEN_BUCKET_*`
- 분석/안전
  - `INPUT_OUTPUT_COUPLING_DETECTED`, `RESPONSE_STRUCTURE_VARIANCE_HIGH`, `STATUS_VARIANCE_DETECTED`
  - `ERROR_DISCLOSURE_PATTERN`, `REQUEST_COUNT`, `TIME_MS`, `ERROR_COUNT`
  - `SCOPE_VIOLATION`, `BLOCKED_DETECTED`, `Errors`
- 플래그
  - `Flag`, `FLAG_FOUND`, `HINT_KEYWORD`, `HINT_BASE64`, `HINT_JWT`, `HINT_COMMENT`
